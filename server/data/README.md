# `server/data/` 数据层说明

本目录封装 JSON 数据文件的读取、写入、规范化和同步逻辑。

## 文件职责

| 文件 | 作用 |
| --- | --- |
| `photoStore.js` | 照片元数据、标签规范化、排序、缩略图路径和照片列表。 |
| `groupStore.js` | 分组扩展数据、封面照片、与照片元数据同步。 |
| `storyStore.js` | 故事数据、故事条目、ID 生成、布局排序和字段规范化。 |

## 数据文件

| 数据文件 | 说明 |
| --- | --- |
| `photo-data.json` | 照片评论、表情、描述、收藏、标签、排序、分组、缩略图。 |
| `group-data.json` | 分组封面等扩展数据。 |
| `story-data.json` | 故事列表、故事条目和故事内容。 |

## 规范化原则

- 读取时兼容空文件、损坏字段和旧数据。
- 写入前统一 normalize。
- 数组字段去空、去重。
- 字符串字段 trim 并限制长度。
- 数字字段校验有限数值。

## 写入策略

数据保存采用：

1. 写入临时文件。
2. rename 覆盖正式文件。

该方式可以降低进程中断导致半文件的风险，但不能解决多进程并发写入。多实例部署前必须增加写入锁或迁移数据库。
