# `server/routes/` 路由说明

本目录保存 HTTP API 路由。路由层负责请求校验、调用数据层/服务层、组装响应和处理错误。

## 路由文件

| 文件 | 挂载路径 | 作用 |
| --- | --- | --- |
| `photos.js` | `/api` | 照片列表、详情、上传、重复检测、批量整理、更新、排序、删除、评论、表情。 |
| `groups.js` | `/api/groups` | 分组创建、重命名、封面设置、删除。 |
| `stories.js` | `/api/stories` | 故事 CRUD、故事条目、故事布局。 |
| `system.js` | `/api/system` | 系统状态、数据健康摘要、打开上传目录。 |

## 设计边界

- 请求体验和错误信息在路由层处理。
- 数据字段默认值和脏数据兼容在 `server/data/` 处理。
- 缩略图生成在 `server/services/thumbnailService.js` 处理。
- 路由不应复制数据规范化逻辑。

## 新增接口清单

新增接口时必须同步：

- `docs/api.md`
- `js/api.js`
- 相关前端业务模块
- 如涉及数据结构，更新 `docs/data-model.md`

## 风险点

- 删除接口会改动真实文件和 JSON 数据。
- 重命名照片会同时影响原图、缩略图、照片元数据和分组封面。
- 故事接口引用照片 ID，照片删除后故事响应会过滤无效照片展示。
