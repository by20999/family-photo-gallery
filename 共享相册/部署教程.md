# 部署到云服务器 - 完整教程

## 方案一：Railway 部署（推荐，最简单）

### 准备工作
1. 注册 GitHub 账号（如果没有）：https://github.com
2. 注册 Railway 账号：https://railway.app （用 GitHub 登录）

### 部署步骤

#### 第一步：初始化 Git 仓库
在 VSCode 终端中运行：
```bash
git init
git add .
git commit -m "初始化相册项目"
```

#### 第二步：推送到 GitHub
1. 在 GitHub 创建新仓库：https://github.com/new
   - 仓库名：`family-photo-gallery`（或任意名字）
   - 设为 Private（私有）
   - 不要勾选任何初始化选项
   - 点击 Create repository

2. 在终端运行（替换成你的 GitHub 用户名）：
```bash
git remote add origin https://github.com/你的用户名/family-photo-gallery.git
git branch -M main
git push -u origin main
```

#### 第三步：在 Railway 部署
1. 访问 https://railway.app
2. 点击 "Start a New Project"
3. 选择 "Deploy from GitHub repo"
4. 授权 Railway 访问你的 GitHub
5. 选择刚才创建的 `family-photo-gallery` 仓库
6. Railway 会自动检测并部署

#### 第四步：获取网址
1. 部署完成后，点击项目
2. 点击 "Settings" 标签
3. 找到 "Domains" 部分
4. 点击 "Generate Domain"
5. 复制生成的网址（类似 `xxx.railway.app`）

**完成！** 把这个网址分享给家人朋友就可以了！

## 方案二：Vercel 部署（需要稍微改造）

Vercel 主要用于静态网站，部署 Node.js 后端需要改成 Serverless 函数。

### 步骤：
1. 访问 https://vercel.com 并用 GitHub 登录
2. 点击 "Add New" → "Project"
3. 导入你的 GitHub 仓库
4. 点击 Deploy

**注意：** Vercel 的免费版有限制，图片会存储在临时文件系统，重启后会丢失。建议用 Railway。

---

## 方案三：阿里云/腾讯云（需要购买服务器）

### 步骤：
1. 购买云服务器（ECS/轻量应用服务器）
2. 安装 Node.js
3. 上传代码到服务器
4. 运行 `npm install` 和 `npm start`
5. 配置域名和防火墙

**费用：** 约 ¥100-300/年

## 常见问题

### Q: Railway 免费吗？
A: 有免费额度，每月 $5 的免费使用额度，足够小型家庭相册使用。

### Q: 图片会丢失吗？
A: Railway 默认使用临时存储，重启会丢失。建议升级使用持久化存储（Volume）或使用对象存储（如阿里云 OSS）。

### Q: 如何绑定自己的域名？
A: 在 Railway 的 Settings → Domains 中可以添加自定义域名。

### Q: 上传的图片有大小限制吗？
A: 目前设置为单张 10MB，一次最多 10 张。可以在 server.js 中修改。

## 推荐配置

如果你想长期使用且不丢失图片，建议：
1. 使用 Railway + 持久化存储（Volume）
2. 或者使用云存储服务（阿里云 OSS、腾讯云 COS）
3. 定期备份 uploads 文件夹
