# 项目计划

## 当前阶段

项目处于 `可用产品 + 工程化补强` 阶段。

已具备：

- 基础相册体验。
- 照片上传与管理。
- 分组、标签、收藏、搜索。
- 评论和表情互动。
- 故事流。
- 文件系统持久化。
- 基础写入鉴权。
- 单进程 API 队列。
- 冒烟测试、数据体检和本地备份脚本。

待补强：

- 系统化测试。
- 备份恢复演练。
- 完整账号体系。
- 错误监控。
- 数据迁移工具。
- 模块拆分。

## 近期计划

1. 完成文档体系。
2. 补齐自动化测试：
   - 数据 normalize 单元测试。
   - 上传、删除、重命名的集成测试。
   - Playwright 浏览器冒烟测试。
3. 演练备份与恢复：
   - 使用 `npm run backup:data` 生成备份。
   - 在临时目录恢复 JSON 数据和 `uploads/`。
   - 记录恢复步骤和耗时。
4. 清理乱码和历史兼容文本。
5. 拆分过大的 `style.css` 和 `js/gallery.js`。

## 中期计划

1. 增加基础登录和家庭成员身份。
2. 增加操作日志。
3. 增加数据恢复入口。
4. 增加对象存储适配层。
5. 增加 CI 检查。

## 长期计划

1. 数据库迁移。
2. 多用户权限。
3. 云端同步。
4. 图片智能分类。
5. 更完整的故事编辑器。

## 暂不计划

- 大型后台管理系统。
- 多租户 SaaS。
- 复杂社交关系链。
- 重型前端框架迁移。

这些方向只有在使用规模明显增长后再重新评估。
