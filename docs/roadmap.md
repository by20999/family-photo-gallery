# 后续方向

## 方向一：可靠性

- 增加备份和恢复脚本。
- 增加 JSON 写入锁或队列，降低并发写入冲突。
- 增加启动时数据体检。
- 增加缩略图重建命令。

## 方向二：安全性

- 替换单一删除密码为登录态。
- 区分查看者、上传者、管理员。
- 为删除评论、删除故事等操作增加权限控制。
- 增加上传文件扩展名和实际内容校验。

## 方向三：体验

- 优化故事流编辑器。
- 增强批量整理界面，例如用专用弹窗替代连续 prompt。
- 增加照片地图或日历视图。
- 重复照片检测基础版已完成，后续增加相似照片检测和重复照片处理界面。
- 增加导出相册包。

## 方向四：工程化

- 增加测试。
- 增加 lint/format。
- 拆分 CSS。
- 拆分 `gallery.js`。
- 引入 CI。

## 方向五：存储升级

- 抽象文件存储接口。
- 支持 S3/R2 等对象存储。
- 支持 SQLite 或 Postgres 存储元数据。
- 提供 JSON 到数据库的迁移脚本。
