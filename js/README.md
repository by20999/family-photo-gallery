# `js/` 前端模块说明

本目录保存浏览器端 ES Module 代码。项目没有前端构建步骤，`index.html` 直接加载 `js/main.js`，再由它初始化其他模块。

## 模块列表

| 文件 | 作用 |
| --- | --- |
| `main.js` | 前端应用入口，负责初始化主题、昵称、评论、删除、灯箱、相册、故事、上传并加载数据。 |
| `api.js` | HTTP 请求封装，所有前端 API 调用应集中在这里。 |
| `state.js` | 全局运行状态和状态更新辅助函数。 |
| `dom.js` | DOM 节点引用集中表。 |
| `gallery.js` | 相册主视图、搜索、标签、分组、排序、批量操作、时间线和首页记忆模块。 |
| `lightbox.js` | 大图灯箱、切图、滤镜、编辑详情、设置分组封面。 |
| `upload.js` | 文件选择、拖拽上传、本地预览、图片压缩和上传进度。 |
| `system.js` | 系统状态面板，展示照片数、缩略图数、数据问题和重复检测结果。 |
| `story.js` | 故事创建、故事选择、故事时间线、拖拽布局和内容编辑。 |
| `comments.js` | 评论和表情回应。 |
| `delete.js` | 删除密码弹窗、单张删除、批量删除和分组删除。 |
| `theme.js` | 明暗模式、渐变背景、主题包和节日提示。 |
| `profile.js` | 昵称、本地用户标识和头像字符。 |
| `feedback.js` | 全局状态提示。 |
| `utils.js` | 通用工具函数。 |

## 依赖方向

- `main.js` 可以依赖业务模块。
- 业务模块可以依赖 `api.js`、`state.js`、`dom.js`、`utils.js`。
- `api.js` 不应依赖 UI 模块。
- `state.js` 不应依赖 DOM。

## 修改约束

- 新增接口请求先加到 `api.js`。
- 新增共享状态先加到 `state.js`。
- 新增 DOM 节点引用先加到 `dom.js`。
- 新增用户可见 HTML 时要注意转义，优先使用 `escapeHtml` 或安全 DOM API。
- 修改 `gallery.js` 时要检查网格、时间线、分组、搜索、标签、收藏、批量模式和上传预览组合。
- 修改 `story.js` 时要同步检查 `server/routes/stories.js` 和 `story-data.json` 数据结构。

## 当前维护风险

- `gallery.js` 体积较大，后续建议拆分为筛选、分组、渲染、批量操作、时间线几个子模块。
- `style.css` 与前端模块耦合较多，改类名时要全局搜索。
- 前端目前没有自动化测试。
